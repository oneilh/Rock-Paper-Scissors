
### Commit When

- Feature is working
- Bug is fixed
- Before risky changes
- Logical stopping point reached
- Tests pass (if applicable)

### Commit Types

| Section           | Suggested Commit Message      |
| :---------------- | :---------------------------- |
| **Feature**       | `feat: short description of what was done`     |
| **Bug Fix**       | `fix: short description of what was fixed`      |
| **Styling/UI Update**       | `style: short description of what was changed`    |
| **Refactor**      | `refactor: short description of what was refactored` |
| **Documentation** | `docs: short description of what was documented`     |
